1、什么是内存池

内存池是一种预分配内存并进行重复利用的技术，通过减少频繁的动态内存分配与释放操作，从而提高程序运行效率。内存池通常预先分配一块大的内存区域，将其划分为多个小块，每次需要分配内存时直接从这块区域中分配，而不是调用系统的动态分配函数==（如new或malloc）==。简单来说就是申请一块较大的内存块(不够继续申请)，之后将这块内存的管理放在==应用层==执行，减少系统调用带来的开销。

理解：预先申请一块较大的内存，之后使用该区域进行内存的分配和回收，减少系统调用带来的开销（全部操作在用户态就可以进行）

前置知识： std::allocator 是 C++ 标准库中的一个类模板，提供了一个通用的接口来管理动态内存的分配和释放。它是标准模板库（STL）的一部分，在诸如 std::vector、std::list 等容器类中被用来管理内存。

主要特点：

  • 通用内存管理：std::allocator 允许容器以类型安全且高效的方式分配和释放内存。
  • 可定制性：你可以通过特化或替换 std::allocator 来提供自己的分配器，从而实现自定义的内存管理策略（例如基于池的分配、垃圾回收等）。
  • 类型特定的分配：分配器由它所管理的类型 T 参数化，确保在分配对象时使用正确的大小和对齐要求。

常见成员函数

  1. allocate(size_t n)：为 n 个类型为 T 的对象分配原始存储空间。该函数不会构造对象，只是保留必要的内存。

  2. deallocate(pointer p, size_t n)：释放指针 p 所指向的存储空间，该指针必须是从之前的 allocate(n) 调用中获得的，并且 n 的值相同。

  3. construct(pointer p, const T& val)：在指针 p 所指向的位置构造一个类型为 T 的对象，并使用值 val 初始化它（通过 placement new）。

  4. destroy(pointer p)：销毁指针 p 所指向的对象，但不释放其内存。

  5. address(reference x)：返回引用 x 所引用的对象的地址，即使 T 类型重载了 operator&。

  6. max_size()：返回可能分配的最大对象数量。

重要点：

  • 分配与构造分离：std::allocator 将内存分配与对象构造分开。allocate() 保留内存，而 construct() 初始化对象。
  • 销毁与释放分离：同样，destroy() 调用对象的析构函数，而 deallocate() 释放内存。
  • 默认分配器：大多数 STL 容器默认使用 std::allocator，除非你指定了自定义分配器。

你可以通过继承 std::allocator 或创建符合 Allocator 概念要求的自定义分配器来实现自定义分配器。自定义分配器可能会覆盖内存池或其他堆区域的行为。

总结
