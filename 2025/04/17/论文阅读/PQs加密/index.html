PQs加密

发表于2025-04-17|更新于2025-04-18|paper reading
|浏览量:

在隐私保护人脸识别（PPFR）框架中，置换、扩散和移位变换是加密算法的核心组成部分。这些变换的目的是通过一系列操作将原始特征向量转换为加密特征向量，从而保护数据的安全性，同时确保加密后的数据仍然可以用于人脸识别任务。以下是对这三种变换作用的详细解释：

1. 置换变换（Permutation Transformation）

作用：

  • 打乱原始数据结构：置换变换通过对特征向量中的元素重新排列顺序，破坏了原始特征向量的结构化信息。
  • 防止直接推断：即使攻击者获得了加密后的特征向量，也无法通过简单的逆向操作恢复原始特征向量的顺序。

  • 置换变换的主要目的是隐藏原始特征向量的排列顺序，使得攻击者无法通过观察加密特征推断出原始特征的结构。

2. 扩散变换（Diffusion Transformation）

作用：

  • 增强随机性和混淆：扩散变换通过引入随机性，使得特征向量中的每个元素都与其他元素发生交互。这样，原始特征向量的一个小变化会导致加密特征向量的多个位置发生变化。
  • 抵抗统计分析攻击：扩散变换使得加密特征向量的统计特性与原始特征向量完全不同，从而防止攻击者通过统计分析推断出原始数据。

实现方式：

  • 使用一个扩散矩阵 (R) 对置换后的特征向量 (x_p) 进行线性变换： [] 其中，(R) 是一个准对角矩阵，由若干正交子矩阵组成。

  • 扩散变换的核心目的是增强加密特征向量的随机性，使得攻击者无法通过局部信息推断出全局信息。

3. 移位变换（Shift Transformation）

作用：

  • 增加常数偏移：移位变换在扩散变换的基础上增加一个随机向量 (s)，进一步增强加密特征向量的随机性。
  • 弥补线性变换的不足：置换和扩散变换本质上是线性变换，虽然能够隐藏原始特征向量的结构，但仍然存在一定的安全风险。移位变换通过引入非线性偏移，进一步提高了加密的安全性。

整体作用

这三种变换共同构成了加密算法的核心流程，通过以下步骤实现隐私保护： 1. 置换变换：打乱原始特征向量的排列顺序。 2. 扩散变换：增强随机性和混淆，使得特征向量的每个元素都与其他元素发生交互。 3. 移位变换：引入随机偏移，进一步增强加密的安全性。

最终生成的加密特征向量 (x_c) 不仅具有高安全性，还保留了原始特征向量的距离特性，从而可以在加密域中进行人脸识别任务。

4. 为什么使用扩散变换？

扩散变换的主要目的是增强加密的安全性，具体体现在以下几个方面： 1. 抗统计分析：扩散变换破坏了原始特征向量的统计特性，使得攻击者无法通过统计方法推断出原始特征向量。 2. 抗差分攻击：扩散变换确保了原始特征向量中任何一个元素的变化都会在加密后的特征向量中引发多个元素的变化，增加了破解难度。 3. 保持距离不变：由于 ( R ) 是正交矩阵，扩散变换不会改变特征向量之间的欧几里得距离。这一点非常重要，因为人脸识别依赖于特征向量之间的距离比较。

5. 扩散变换的实际效果

这一结果表明，扩散变换在增强安全性的同时，不会影响人脸识别的准确性。

6. 总结

矩阵 ( R ) 是扩散变换的核心组件，其作用是通过对特征向量进行正交变换来扰乱原始特征向量的结构。( R ) 是一个准对角矩阵，由多个正交矩阵组成，具有正交矩阵的范数不变性和线性组合的特性。扩散变换不仅增强了加密的安全性，还保持了特征向量之间的距离不变，从而确保了加密后的特征向量仍然可以用于人脸识别任务。
