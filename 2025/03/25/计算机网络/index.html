计算机网络笔记

发表于2025-03-25|更新于2025-03-25|专业课复习
|浏览量:

一、计算级网络的功能 [[#1. 数据通信]]

1. 数据通信

  • 作用 ：将数据从一个节点传输到另一个节点。
  • 例子 ：
    • 电子邮件（Email）发送与接收。
    • 文件传输（如FTP）。
    • 即时通讯（如微信、QQ） #### 2. 资源共享

资源共享是计算机网络的核心目标之一，旨在高效利用有限的资源。

  • 硬件资源共享 ：
    • 共享打印机、存储设备等。
    • 云计算中的计算资源（CPU、内存）共享。
  • 软件资源共享 ：
    • 共享应用程序、数据库等。
  • 数据资源共享 ：
    • 共享文件、文档、图片等。 #### 3. 分布式处理

分布式处理指将任务分解为多个子任务，分配到不同的计算机上并行处理。

  • 优势 ：
    • 提高计算效率。
    • 增强系统的可靠性和容错能力。
  • 例子 ：
    • 分布式数据库（如Hadoop）。
    • 区块链技术中的分布式账本。

4. 可靠性与容错性

网络设计中通常考虑冗余机制，以提高系统的可靠性。

  • 作用 ：
    • 当某个节点或链路发生故障时，可以通过备用路径继续通信。
  • 例子 ：
    • 冗余路由器、负载均衡器。
    • 网络备份与恢复机制。

二、计算级网络的分类

计算机网络可以根据不同的标准进行分类，例如覆盖范围、拓扑结构、传输介质、管理方式等。以下是对计算机网络分类的系统化整理：

一、按覆盖范围分类

根据网络覆盖的地理范围，可以将计算机网络分为以下几类：

1. 局域网（LAN, Local Area Network）

  • 特点：
    • 覆盖范围较小（通常在几十米到几公里内）。
    • 高速数据传输（如100Mbps~10Gbps）。
    • 使用专用通信线路（如以太网、Wi-Fi）。
  • 应用场景：
    • 家庭网络、办公室网络。
    • 校园网、企业内部网。
  • 例子：
    • 使用交换机和路由器连接多台电脑组成的网络。

2. 城域网（MAN, Metropolitan Area Network）

  • 特点：
    • 覆盖范围较大（一个城市或地区，通常为5~50公里）。
    • 数据传输速率较高。
    • 多用于公共服务（如政府、教育机构之间的互联）。
  • 应用场景：
    • 城市内的宽带接入网络。
    • 电信运营商提供的城域网服务。
  • 例子：
    • 通过光纤连接多个局域网形成的城市级网络。

3. 广域网（WAN, Wide Area Network）

  • 特点：
    • 覆盖范围广（跨城市、国家甚至全球）。
    • 数据传输速率较低（相比LAN）。
    • 使用公共通信设施（如电话线、卫星、光纤）。
  • 应用场景：
    • 互联网（Internet）。
    • 跨国企业的分支机构互联。
  • 例子：
    • 全球范围内的云计算服务。

4. 个域网（PAN, Personal Area Network）

  • 特点：
    • 覆盖范围极小（几米范围内）。
    • 主要用于个人设备之间的通信。
  • 应用场景：
    • 蓝牙耳机与手机的连接。
    • 智能手表与手机的数据同步。
  • 例子：
    • 使用蓝牙技术连接的设备网络。

二、按拓扑结构分类

网络拓扑结构是指网络中节点（如计算机、路由器）和通信链路的物理或逻辑布局。

1. 星型拓扑（Star Topology）

  • 特点：
    • 所有节点通过单独的链路连接到一个==中心节点==（如交换机或集线器）。
    • 中心节点故障会导致整个网络瘫痪。
  • 优点：
    • 易于管理和扩展。
    • 单点故障影响较小（除中心节点外）。
  • 应用场景：
    • 办公室局域网。

2. 总线型拓扑（Bus Topology）

  • 特点：
    • 所有节点共享一条主干通信线路（总线）。
    • 数据沿总线传播，所有节点都能接收到。
  • 优点：
    • 结构简单，成本低。
  • 缺点：
    • 总线故障会导致整个网络失效。
  • 应用场景：
    • 小型网络（现已较少使用）。

3. 环型拓扑（Ring Topology）

  • 特点：
    • 节点通过通信链路组成一个闭合环。
    • 数据沿环单向或双向传输。
  • 优点：
    • 数据传输有序，冲突较少。
  • 缺点：
    • 环上任意一点故障会影响整个网络。
  • 应用场景：
    • FDDI（光纤分布式数据接口）网络。

4. 网状拓扑（Mesh Topology）

  • 特点：
    • 每个节点与其他节点之间有多条链路直接连接。
  • 优点：
    • 冗余性强，可靠性高。
  • 缺点：
    • 成本高，布线复杂。
  • 应用场景：
    • 数据中心网络、骨干网。

5. 树型拓扑（Tree Topology）

  • 特点：
    • 类似星型拓扑的扩展，呈层级结构。
    • 适用于大规模网络。
  • 优点：
    • 易于扩展和管理。
  • 缺点：
    • 根节点故障会影响整个网络。
  • 应用场景：
    • 校园网、企业网络。

三、按传输介质分类

根据数据传输所使用的物理介质，网络可以分为以下几类：

1. 有线网络

  • 特点：
    • 使用物理介质（如铜线、光纤）传输数据。
  • 优点：
    • 传输速率高，稳定性好。
  • 缺点：
    • 布线成本高，移动性差。
  • 常见类型：
    • 双绞线（如Cat5e、Cat6）。
    • 同轴电缆。
    • 光纤。

2. 无线网络

  • 特点：
    • 使用电磁波（如无线电波、微波）传输数据。
  • 优点：
    • 移动性强，部署灵活。
  • 缺点：
    • 易受干扰，安全性较低。
  • 常见类型：
    • Wi-Fi（IEEE 802.11）。
    • 蓝牙（Bluetooth）。
    • 4G/5G蜂窝网络。

四、按管理方式分类（主机间的通信方式）

1. 对等网络（Peer-to-Peer, P2P）

  • 特点：
    • 网络中的每个节点既是客户端又是服务器。
    • 资源和服务由节点之间直接共享。
  • 优点：
    • 无需专用服务器，成本低。
  • 缺点：
    • 安全性较低，管理复杂。
  • 应用场景：
    • 文件共享（如BitTorrent）。
    • 区块链网络。

2. 客户端/服务器网络（Client/Server）

  • 特点：
    • 网络中存在专用的服务器提供服务，客户端请求服务。
  • 优点：
    • 管理集中，安全性高。
  • 缺点：
    • 服务器成为单点故障。
  • 应用场景：
    • 企业内部网、Web服务。

五、按用途分类

1. 专用网络

  • 特点：
    • 专为特定组织或目的设计。
    • 不对外公开。
  • 例子：
    • 军用网络、银行内部网络。

2. 公共网络

  • 特点：
    • 面向公众开放。
    • 提供广泛的服务。
  • 例子：
    • 互联网（Internet）。

六、总结

计算机网络的分类可以从多个维度进行，以下是简要总结：

分类标准 类别 特点
覆盖范围 LAN、MAN、WAN、PAN 从小范围到全球范围
拓扑结构 星型、总线型、环型、网状、树型 不同的物理或逻辑布局
传输介质 有线、无线 使用不同的物理介质
管理方式 P2P、客户端/服务器 资源共享方式不同
用途 专用网络、公共网络 面向特定用户或公众
