计算机网络笔记

|浏览量:

引言

本文为我复习计算机机网络过程，大部分内容由AI生成，如有错误请见谅，思维导图是我复习408的时候做的。

一、计算级网络的功能

1. 数据通信

  • 作用 ：将数据从一个节点传输到另一个节点。
  • 例子 ：
    • 电子邮件（Email）发送与接收。
    • 文件传输（如FTP）。
    • 即时通讯（如微信、QQ）

2. 资源共享

资源共享是计算机网络的核心目标之一，旨在高效利用有限的资源。

  • 硬件资源共享 ：
    • 共享打印机、存储设备等。
    • 云计算中的计算资源（CPU、内存）共享。
  • 软件资源共享 ：
    • 共享应用程序、数据库等。
  • 数据资源共享 ：
    • 共享文件、文档、图片等。 z #### 3. 分布式处理

分布式处理指将任务分解为多个子任务，分配到不同的计算机上并行处理。

  • 优势 ：
    • 提高计算效率。
    • 增强系统的可靠性和容错能力。
  • 例子 ：
    • 分布式数据库（如Hadoop）。
    • 区块链技术中的分布式账本。

4. 可靠性与容错性

网络设计中通常考虑冗余机制，以提高系统的可靠性。

  • 作用 ：
    • 当某个节点或链路发生故障时，可以通过备用路径继续通信。
  • 例子 ：
    • 冗余路由器、负载均衡器。
    • 网络备份与恢复机制。

二、计算级网络的分类

计算机网络可以根据不同的标准进行分类，例如覆盖范围、拓扑结构、传输介质、管理方式等。以下是对计算机网络分类的系统化整理：

一、按覆盖范围分类

根据网络覆盖的地理范围，可以将计算机网络分为以下几类：

1. 局域网（LAN, Local Area Network）

  • 特点：
    • 覆盖范围较小（通常在几十米到几公里内）。
    • 高速数据传输（如100Mbps~10Gbps）。
    • 使用专用通信线路（如以太网、Wi-Fi）。
  • 应用场景：
    • 家庭网络、办公室网络。
    • 校园网、企业内部网。
  • 例子：
    • 使用交换机和路由器连接多台电脑组成的网络。

2. 城域网（MAN, Metropolitan Area Network）

  • 特点：
    • 覆盖范围较大（一个城市或地区，通常为5~50公里）。
    • 数据传输速率较高。
    • 多用于公共服务（如政府、教育机构之间的互联）。
  • 应用场景：
    • 城市内的宽带接入网络。
    • 电信运营商提供的城域网服务。
  • 例子：
    • 通过光纤连接多个局域网形成的城市级网络。

3. 广域网（WAN, Wide Area Network）

  • 特点：
    • 覆盖范围广（跨城市、国家甚至全球）。
    • 数据传输速率较低（相比LAN）。
    • 使用公共通信设施（如电话线、卫星、光纤）。
  • 应用场景：
    • 互联网（Internet）。
    • 跨国企业的分支机构互联。
  • 例子：
    • 全球范围内的云计算服务。

4. 个域网（PAN, Personal Area Network）

  • 特点：
    • 覆盖范围极小（几米范围内）。
    • 主要用于个人设备之间的通信。
  • 应用场景：
    • 蓝牙耳机与手机的连接。
    • 智能手表与手机的数据同步。
  • 例子：
    • 使用蓝牙技术连接的设备网络。

二、按拓扑结构分类

网络拓扑结构是指网络中节点（如计算机、路由器）和通信链路的物理或逻辑布局。

1. 星型拓扑（Star Topology）

  • 特点：
    • 所有节点通过单独的链路连接到一个==中心节点==（如交换机或集线器）。
    • 中心节点故障会导致整个网络瘫痪。
  • 优点：
    • 易于管理和扩展。
    • 单点故障影响较小（除中心节点外）。
  • 应用场景：
    • 办公室局域网。

2. 总线型拓扑（Bus Topology）

  • 特点：
    • 所有节点共享一条主干通信线路（总线）。
    • 数据沿总线传播，所有节点都能接收到。
  • 优点：
    • 结构简单，成本低。
  • 缺点：
    • 总线故障会导致整个网络失效。
  • 应用场景：
    • 小型网络（现已较少使用）。

3. 环型拓扑（Ring Topology）

  • 特点：
    • 节点通过通信链路组成一个闭合环。
    • 数据沿环单向或双向传输。
  • 优点：
    • 数据传输有序，冲突较少。
  • 缺点：
    • 环上任意一点故障会影响整个网络。
  • 应用场景：
    • FDDI（光纤分布式数据接口）网络。

4. 网状拓扑（Mesh Topology）

  • 特点：
    • 每个节点与其他节点之间有多条链路直接连接。
  • 优点：
    • 冗余性强，可靠性高。
  • 缺点：
    • 成本高，布线复杂。
  • 应用场景：
    • 数据中心网络、骨干网。

5. 树型拓扑（Tree Topology）

  • 特点：
    • 类似星型拓扑的扩展，呈层级结构。
    • 适用于大规模网络。
  • 优点：
    • 易于扩展和管理。
  • 缺点：
    • 根节点故障会影响整个网络。
  • 应用场景：
    • 校园网、企业网络。

三、按传输介质分类

根据数据传输所使用的物理介质，网络可以分为以下几类：

1. 有线网络

  • 特点：
    • 使用物理介质（如铜线、光纤）传输数据。
  • 优点：
    • 传输速率高，稳定性好。
  • 缺点：
    • 布线成本高，移动性差。
  • 常见类型：
    • 双绞线（如Cat5e、Cat6）。
    • 同轴电缆。
    • 光纤。

2. 无线网络

  • 特点：
    • 使用电磁波（如无线电波、微波）传输数据。
  • 优点：
    • 移动性强，部署灵活。
  • 缺点：
    • 易受干扰，安全性较低。
  • 常见类型：
    • Wi-Fi（IEEE 802.11）。
    • 蓝牙（Bluetooth）。
    • 4G/5G蜂窝网络。

四、按管理方式分类（主机间的通信方式）

1. 对等网络（Peer-to-Peer, P2P）

  • 特点：
    • 网络中的每个节点既是客户端又是服务器。
    • 资源和服务由节点之间直接共享。
  • 优点：
    • 无需专用服务器，成本低。
  • 缺点：
    • 安全性较低，管理复杂。
  • 应用场景：
    • 文件共享（如BitTorrent）。
    • 区块链网络。

2. 客户端/服务器网络（Client/Server）

  • 特点：
    • 网络中存在专用的服务器提供服务，客户端请求服务。
  • 优点：
    • 管理集中，安全性高。
  • 缺点：
    • 服务器成为单点故障。
  • 应用场景：
    • 企业内部网、Web服务。

五、按用途分类

1. 专用网络

  • 特点：
    • 专为特定组织或目的设计。
    • 不对外公开。
  • 例子：
    • 军用网络、银行内部网络。

2. 公共网络

  • 特点：
    • 面向公众开放。
    • 提供广泛的服务。
  • 例子：
    • 互联网（Internet）。

六、总结

计算机网络的分类可以从多个维度进行，以下是简要总结：

分类标准 类别 特点
覆盖范围 LAN、MAN、WAN、PAN 从小范围到全球范围
拓扑结构 星型、总线型、环型、网状、树型 不同的物理或逻辑布局
传输介质 有线、无线 使用不同的物理介质
管理方式 P2P、客户端/服务器 资源共享方式不同
用途 专用网络、公共网络 面向特定用户或公众

四、计算机网络的组成

计算机网络是由多个硬件、软件和协议组成的复杂系统，用于实现不同设备之间的通信和资源共享。为了清晰地理解计算机网络的组成，可以从以下几个方面进行分析：==硬件组成、软件组成、协议体系以及逻辑结构==。

1. 硬件组成

硬件是计算机网络的基础，主要包括以下几类设备：

（1）终端设备（End Devices）

  • 定义：终端设备是网络中的用户设备，用于生成、发送或接收数据。
  • 常见设备：
    • 计算机（PC、笔记本电脑）
    • 智能手机、平板电脑
    • 打印机、扫描仪
    • IoT设备（如智能家电、传感器）

（2）网络设备（Networking Devices）

  • 定义：网络设备负责数据的传输、路由和交换。
  • 常见设备：
    • 网卡（NIC, Network Interface Card）：连接设备与网络的接口。
    • 交换机（Switch）：在局域网中用于数据帧的转发。
    • 路由器（Router）：在不同网络之间转发数据包。
    • 集线器（Hub）：早期的网络设备，用于广播数据（已逐渐被淘汰）。
    • 无线接入点（AP, Access Point）：提供无线网络连接。
    • 调制解调器（Modem）：将数字信号转换为模拟信号（反之亦然），用于广域网接入。

（3）传输介质（Transmission Media）

  • 定义：传输介质是数据在网络中传输的物理载体。
  • 分类：
    • 有线介质：
      • 双绞线（Twisted Pair Cable）
      • 同轴电缆（Coaxial Cable）
      • 光纤（Fiber Optic Cable）
    • 无线介质：
      • 无线电波（Radio Waves）
      • 微波（Microwave）
      • 红外线（Infrared）
      • 卫星通信

2. 软件组成

软件是计算机网络的灵魂，负责管理和控制网络的运行。主要包括以下几类：

（1）网络操作系统（NOS, Network Operating System）

  • 定义：网络操作系统是管理网络资源和提供服务的软件平台。
  • 示例：
    • Windows Server
    • Linux（如Ubuntu Server、CentOS）
    • Cisco IOS（专用于网络设备）

（2）网络应用软件

  • 定义：用于实现具体的网络功能和服务。
  • 示例：
    • 浏览器（如Chrome、Firefox）
    • 文件传输工具（如FTP客户端）
    • 邮件客户端（如Outlook、Thunderbird）
    • 视频会议软件（如Zoom、Teams）

（3）网络管理软件

  • 定义：用于监控、配置和维护网络。
  • 示例：
    • SNMP（Simple Network Management Protocol）工具
    • 网络监控工具（如Wireshark、PRTG）

3. 协议体系

协议是计算机网络的核心，定义了数据传输的规则和标准。协议体系通常分为分层模型，其中最常用的是OSI七层模型和TCP/IP四层模型。

（1）OSI七层模型

==结构分明，常用于教学== - 定义：国际标准化组织（ISO）提出的开放式系统互联参考模型，分为七层。 - 层次结构： 1. 物理层（Physical Layer）：负责比特流的传输。 2. 数据链路层（Data Link Layer）：负责帧的传输和错误检测。 3. 网络层（Network Layer）：负责数据包的路由和转发。 4. 传输层（Transport Layer）：负责端到端的可靠传输。 5. 会话层（Session Layer）：负责建立、管理和终止会话。 6. 表示层（Presentation Layer）：负责数据格式转换和加密。 7. 应用层（Application Layer）：提供用户接口和服务。

（2）TCP/IP四层模型

  • 定义：==实际应用==中最常用的协议模型，分为四层。
  • 层次结构：
    1. 网络接口层（Network Interface Layer）：对应OSI的物理层和数据链路层。
    2. 网络层（Internet Layer）：负责IP地址和路由，主要协议是IP。
    3. 传输层（Transport Layer）：负责端到端通信，主要协议是TCP和UDP。
    4. 应用层（Application Layer）：提供用户服务，如HTTP、FTP、SMTP等。 后续会详细介绍每一层以及两种模型的区别

（3）二者相似之处与区别

同：二者都采用层分离结构，都能实现==网络异构== 异：OSI网络层支持 无连接和面向连接，传输层仅面向连接，TCP/IP网络层无连接，传输层支持无连接和面向连接

==协议三要素==

协议是计算机网络中通信双方需要共同遵守的规则和约定。为了确保通信的顺利进行，协议必须包含三个基本要素，即语法、语义和时序。这三要素共同定义了数据如何传输、解释以及何时传输。

1. 语法（Syntax）

  • 定义：语法规定了数据的格式、结构和编码方式。
  • 作用：它定义了通信双方交换的数据应该如何组织和表示。
  • 具体内容：
    • 数据的格式（如帧、包、段等）。
    • 数据的编码方式（如ASCII、UTF-8）。
    • 数据字段的排列顺序和长度。
  • 示例：
    • 在HTTP协议中，请求消息的语法包括请求行、头部字段和消息体。
    • 在以太网帧中，语法规定了帧头、数据部分和帧尾的结构。

2. 语义（Semantics）

  • 定义：语义规定了通信双方需要发送的控制信息、操作的意义以及对特定事件的响应方式。
  • 作用：它定义了通信双方如何理解和解释数据。
  • 具体内容：
    • 控制信息的含义（如命令、状态码）。
    • 错误检测和处理机制。
    • 数据字段的具体意义。
  • 示例：
    • 在TCP协议中，SYN、ACK、FIN标志位分别表示建立连接、确认接收和断开连接。
    • 在HTTP协议中，状态码（如200表示成功，404表示未找到资源）属于语义范畴。

3. 时序（Timing/Synchronization）

  • 定义：时序规定了事件发生的顺序以及通信的速度匹配问题。
  • 作用：它定义了通信双方在时间上的协调关系。
  • 具体内容：
    • 数据发送和接收的时间安排。
    • 流量控制和拥塞控制机制。
    • 超时重传机制。
  • 示例：
    • 在TCP协议中，三次握手的过程严格遵循时序要求：客户端先发送SYN，服务器回复SYN+ACK，客户端再发送ACK。
    • 在滑动窗口协议中，发送方和接收方通过窗口大小来协调数据传输速率。

总结

协议的三要素可以总结如下：

要素 定义 作用 示例
语法 数据的格式和结构 规范数据的组织方式 HTTP请求格式、以太网帧结构
语义 控制信息的含义和操作的意义 定义数据的解释和处理方式 TCP标志位、HTTP状态码
时序 事件的顺序和时间协调 确保通信双方的时间同步和速率匹配 TCP三次握手、滑动窗口机制

4. 逻辑结构

从逻辑上看，计算机网络可以分为以下几个部分：

（1）资源子网（高三层）

  • 定义：资源子网是网络中提供资源和服务的部分，主要由终端设备和服务器组成。
  • 功能：
    • 提供数据存储、计算能力。
    • 提供网络服务（如Web服务、文件共享）。

（2）通信子网（低三层）

  • 定义：通信子网是网络中负责数据传输的部分，主要由网络设备和传输介质组成。
  • 功能：
    • 数据的传输、路由和交换。
    • 实现设备之间的互联互通。

五、交换方式

计算机网络中的交换方式是指数据在网络中从源节点传输到目标节点时所采用的路径选择和转发方法。不同的交换方式适用于不同的场景，具有各自的优缺点。以下是计算机网络中常见的交换方式及其特点：

1. 电路交换（Circuit Switching）

（1）定义

电路交换是一种在通信开始前建立专用物理连接的方式。在通信过程中，该连接始终被独占，直到通信结束。传输单位是比特

（2）工作原理

  • 在通信开始之前，源节点和目标节点之间需要建立一条专用的物理通路。
  • 数据通过这条专用通路进行传输，通信结束后释放连接。

（3）优点

  • 低延迟：一旦连接建立，数据传输几乎没有延迟。
  • 高可靠性：专用连接保证了稳定的通信质量。
  • 实时性强：适合语音和视频等实时性要求高的应用。

（4）缺点

  • 资源利用率低：即使没有数据传输，连接仍然占用资源。
  • 建立连接耗时：需要时间来建立和拆除连接。
  • 灵活性差：无法动态调整路径。

（5）应用场景

  • 传统电话网络（PSTN）。
  • 实时通信系统（如语音通话、视频会议）。

2. 报文交换（Message Switching）

（1）定义

报文交换是一种存储转发的交换方式。数据被封装成一个完整的报文，在每个中间节点上存储并转发。传输单位是报文

（2）工作原理

  • 源节点将整个报文发送到第一个中间节点。
  • 中间节点接收并存储整个报文，然后根据路由信息（查询路由表）转发到下一个节点。
  • 报文最终到达目标节点。

（3）优点

  • 资源利用率高：不需要专用连接，多个报文可以共享链路。
  • 灵活路由：可以根据网络状况动态选择最佳路径。

（4）缺点

  • 延迟较高：由于需要存储整个报文，转发速度较慢。
  • 内存需求大：中间节点需要足够的存储空间来缓存报文。
  • 不适合实时通信：延迟较高，不适合语音或视频等实时应用。

（5）应用场景

  • 早期的电报网络。
  • 现代网络中较少使用，逐渐被分组交换取代。

3. 分组交换（Packet Switching）

（1）定义

分组交换是一种将数据分割成小块（称为分组或数据包），并通过网络独立传输的方式。传输单位是报文段

（2）分类

分组交换分为两种主要类型： - 数据报方式（Datagram）： - 每个分组独立传输，可能通过不同的路径到达目标节点。 - 不需要预先建立连接。 - ==虚电路方式==（Virtual Circuit）： - 在通信开始前，先建立一条==逻辑==连接（虚电路）。 - 所有分组沿着相同的路径传输。 - 某个节点故障，所有经过该节点的虚电路都将遭到破坏 - 可靠，按序到达 - 仅在建立连接是使用完整地址、后续仅使用虚电路号

（3）工作原理

  • 源节点将数据分割成多个分组，每个分组包含目标地址和序号。
  • 分组在网络中逐跳传输，中间节点根据路由表转发分组。
  • 目标节点接收到所有分组后，按序重组为原始数据。

（4）优点

  • 高效利用资源：多个分组可以共享链路，提高带宽利用率。
  • 灵活性高：支持动态路由选择，适应网络变化。
  • 适合多种应用：既能满足实时性要求，也能处理大数据量传输。

（5）缺点

  • 延迟不固定：分组可能通过不同路径到达目标节点，导致延迟不一致。
  • 复杂性高：需要管理分组的排序和重组。
  • 需要设置标记，增大开销。

（6）应用场景

  • 现代互联网（基于IP协议）。
  • 数据传输、文件共享、在线服务等。

4. 三种交换方式的对比

特性 电路交换 报文交换 分组交换
资源分配 专用连接 存储转发 动态共享
延迟 低（建立连接后无额外延迟） 高（需存储完整报文） 中（取决于分组大小和路由）
资源利用率 低（独占资源） 高（无需专用连接） 高（动态共享资源）
实时性 强（适合实时通信） 弱（延迟高） 中（可调优以适应实时通信）
典型应用 电话网络 电报网络 互联网

5. 总结

  • 电路交换适合实时性要求高的通信场景，但资源利用率较低。
  • 报文交换因延迟较高且效率较低，已逐渐被淘汰。
  • 分组交换是现代计算机网络的核心技术，具有高效、灵活的特点，广泛应用于互联网和其他数据通信场景。

六、计算机网络性能指标

计算机网络的性能指标是用来衡量网络运行效率、可靠性和服务质量的关键参数。这些指标对于评估网络性能、优化网络设计以及解决网络问题具有重要意义。以下是计算机网络中常见的性能指标及其详细解释：

1. 带宽（Bandwidth）

（1）定义

带宽是指网络链路在单位时间内能够传输的最大数据量，通常以比特每秒（bps）为单位。

（2）特点

  • 带宽是网络链路的理论==最大容量==。
  • 实际传输速率可能低于带宽，受多种因素（如协议开销、拥塞等）影响。

（3）单位

  • 常见单位：Kbps（千比特每秒）、Mbps（兆比特每秒）、Gbps（千兆比特每秒）。

（4）应用场景

  • 用于描述网络链路的能力，例如家庭宽带的下载速度。

2. 吞吐量（Throughput）

（1）定义

吞吐量是指网络在实际使用中单位时间内成功传输的数据量，通常也以bps为单位。

（2）特点

  • 吞吐量反映了网络的==实际性能==，可能低于带宽。
  • 受到网络拥塞、协议效率和设备性能的影响。

（3）与带宽的区别

  • 带宽是理论值，表示链路的最大容量。
  • 吞吐量是实际值，表示实际传输的数据量。

（4）应用场景

  • 测量文件传输速度或视频流媒体的实际速率。

3. 时延（Latency/Delay）

（1）定义

时延是指数据从源节点发送到目标节点所需的时间，通常以毫秒（ms）为单位。

（2）组成部分

  • 发送时延（Transmission Delay）：将数据推送到链路上所需的时间，取决于数据大小和链路带宽。

  • 传播时延（Propagation Delay）：数据在物理介质上传播所需的时间，取决于距离和信号传播速度。

  • 处理时延（Processing Delay）：中间节点处理数据包所需的时间。

  • 排队时延（Queuing Delay）：数据包在网络设备队列中等待的时间。

    ==时延带宽积 = 传播时延信道带宽==

（3）应用场景

  • 衡量实时通信（如语音通话、在线游戏）的质量。

4. 往返时间（Round-Trip Time, RTT）

（1）定义

RTT是指数据包从源节点发送到目标节点并返回所需的时间。

（2）特点

  • RTT包括发送时延、传播时延和处理时延。
  • 是衡量网络延迟的重要指标。

（3）应用场景

  • 测试网络连接质量（如ping命令测量RTT）。

5. 丢包率（Packet Loss Rate）

（1）定义

丢包率是指在网络传输过程中丢失的数据包占总发送数据包的比例，通常以百分比表示。

（2）原因

  • 网络拥塞。
  • 设备故障或链路中断。
  • 数据包损坏或超时未到达。

（3）影响

  • 对实时应用（如语音、视频）影响较大，可能导致卡顿或失真。

（4）应用场景

  • 监控网络稳定性，检测拥塞或故障。

6. 抖动（Jitter）

（1）定义

抖动是指数据包到达时间的不一致性，即相邻数据包之间时延的变化。

（2）影响

  • 对实时通信（如语音、视频）影响显著，可能导致音视频不同步或卡顿。

（3）应用场景

  • 测量实时流媒体服务的质量。

7. 利用率（Utilization）

（1）定义

利用率是指网络资源（如链路、带宽）被实际使用的比例。

（2）计算公式

（3）特点

  • 高利用率可能导致网络拥塞。
  • 过低的利用率可能意味着资源浪费。

（4）应用场景

  • 评估网络资源的使用效率。

总结

以下是计算机网络性能指标的分类和对比：

指标 定义 单位/形式 应用场景
带宽 链路的最大数据传输能力 bps 描述链路能力
吞吐量 实际传输的数据量 bps 测量实际性能
时延 数据从源到目标所需的时间 毫秒（ms） 实时通信质量评估
RTT 数据包往返所需的时间 毫秒（ms） 网络连接质量测试
丢包率 数据包丢失的比例 百分比（%） 网络稳定性监控

八、编码和调制

编码和调制是计算机网络和通信系统中两个重要的概念，它们分别用于数据的数字化表示和信号的传输。虽然两者密切相关，但它们的作用和应用场景有所不同。

1. 编码（Encoding）

（1）定义

编码是指将数字数据转换为适合在信道上传输的信号形式的过程。它主要关注如何将二进制数据（0和1）映射到特定的信号模式。

（2）作用

  • 将原始数据转换为能在物理介质上传输的形式。
  • 提高数据传输的可靠性，减少错误。
  • 在某些情况下，通过编码实现数据压缩或加密。

（3）分类

编码可以分为以下几类：

a. 数字编码

数字编码是指将二进制数据转换为数字信号的过程，主要用于基带传输（如局域网中的以太网）。 - 常见数字编码方式： - NRZ（Non-Return-to-Zero）：用高电平表示1，低电平表示0。 - 曼彻斯特编码（Manchester Encoding）：每个比特中间有一个跳变，既表示数据又提供时钟同步。 - 差分曼彻斯特编码（Differential Manchester Encoding）：跳变的位置表示数据，而不是电平本身。 - 4B/5B编码：将4位数据编码为5位符号，减少直流分量并提高传输效率。

b. 模拟编码

模拟编码是指将数字数据转换为模拟信号的过程，主要用于频带传输（如电话线上的数据传输）。 - 常见模拟编码方式： - ASK（Amplitude Shift Keying，幅移键控）：通过改变载波信号的幅度来表示数据。 - FSK（Frequency Shift Keying，频移键控）：通过改变载波信号的频率来表示数据。 - PSK（Phase Shift Keying，相移键控）：通过改变载波信号的相位来表示数据。

（4）应用场景

  • 局域网中的以太网使用曼彻斯特编码。
  • 光纤通信中使用NRZ或8B/10B编码。
  • 调制解调器（Modem）使用ASK、FSK等模拟编码技术。

2. 调制（Modulation）

（1）定义

调制是指将数字信号或模拟信号加载到高频载波信号上的过程。通过调制，信号可以在无线或有线信道中高效传输。

（2）作用

  • 将信号的频谱搬移到更高的频率范围，以便适应信道特性。
  • 提高信号的抗干扰能力。
  • 实现多路复用（如频分复用FDM、正交频分复用OFDM）。

（3）分类

调制可以分为以下几类：

a. 模拟调制

模拟调制是指将模拟信号加载到载波信号上的过程，主要用于传统的广播和电视信号传输。 - 常见模拟调制方式： - AM（Amplitude Modulation，幅度调制）：通过改变载波信号的幅度来表示信息。 - FM（Frequency Modulation，频率调制）：通过改变载波信号的频率来表示信息。 - PM（Phase Modulation，相位调制）：通过改变载波信号的相位来表示信息。

b. 数字调制

数字调制是指将数字信号加载到载波信号上的过程，主要用于现代无线通信（如Wi-Fi、蜂窝网络）。 - 常见数字调制方式： - ASK（Amplitude Shift Keying，幅移键控）：通过改变载波信号的幅度来表示0和1。 - FSK（Frequency Shift Keying，频移键控）：通过改变载波信号的频率来表示0和1。 - PSK（Phase Shift Keying，相移键控）：通过改变载波信号的相位来表示0和1。 - QAM（Quadrature Amplitude Modulation，正交幅度调制）：同时改变载波信号的幅度和相位，实现更高的数据传输速率。

（4）应用场景

  • 广播电台使用AM或FM调制。
  • Wi-Fi和4G/5G网络使用QAM调制。
  • 卫星通信中使用PSK或QAM调制。

3. 编码与调制的区别

特性 编码 调制
作用 将数据转换为适合传输的信号形式 将信号加载到载波上以便在信道中传输
输入 数字数据（0和1） 已编码的数字信号或模拟信号
输出 数字信号或模拟信号 已调制的高频信号
应用场景 基带传输（如以太网）、光纤通信 频带传输（如无线通信、卫星通信）
典型技术 NRZ、曼彻斯特编码、4B/5B编码 ASK、FSK、PSK、QAM

4. 总结

  • 编码主要用于将数据转换为适合传输的信号形式，关注的是信号的表示方式。
  • 调制主要用于将信号加载到载波上，关注的是信号的传输效率和抗干扰能力。

两者通常结合使用：先对数据进行编码，然后通过调制将其加载到载波信号上进行传输。例如，在Wi-Fi中，数据首先通过数字编码（如QAM）表示，然后通过调制技术加载到射频信号上进行无线传输。

九、多用户共享信道的技术

频分复用（FDM）、时分复用（TDM）、波分复用（WDM）和码分复用（CDM）是通信系统中用于在共享信道上实现多路信号传输的技术，属于多路复用技术（Multiplexing Technology）。它们的核心目标是允许多个信号在同一物理信道上传输，但通过不同的方式区分信号。以下从定义、原理、特点和应用场景等方面详细对比这四种技术。

1. 频分复用（FDM, Frequency Division Multiplexing）

（1）定义

频分复用是一种将信道的可用带宽划分为多个不重叠的频段，每个频段分配给一个信号进行传输的技术。

（2）工作原理

  • 整个频谱被分割成若干个子频段。
  • 每个信号占用一个独立的频段，同时进行传输。
  • 在接收端，通过滤波器分离出不同频段的信号。

（3）特点

  • 优点：
    • 实现简单，技术成熟。
    • 适合模拟信号和数字信号的传输。
  • 缺点：
    • 频谱利用率较低，尤其是在用户数量较多时。
    • 动态分配频谱的能力较差。

（4）应用场景

  • 广播电台和电视台（如AM/FM广播）。
  • 传统的电话网络（如DSL技术）。
  • 无线通信系统（如第一代蜂窝网络AMPS）。

2. 时分复用（TDM, Time Division Multiplexing）

（1）定义

时分复用是一种将时间划分为多个时隙，每个信号在分配的时隙内独占信道进行传输的技术。

（2）工作原理

  • 时间轴被分成等长的时隙，每个信号依次使用一个时隙。
  • 所有信号轮流使用信道，但在接收端按照时间顺序还原信号。

（3）特点

  • 优点：
    • 频谱利用率较高，适合数字信号传输。
    • 可扩展性强，支持动态分配时隙。
  • 缺点：
    • 对同步要求高，需要精确的时间控制。
    • 如果某个信号未使用其分配的时隙，会造成资源浪费。

（4）应用场景

  • 数字电话网络（如ISDN、E1/T1线路）。
  • 计算机网络中的数据传输（如以太网帧）。
  • 第二代移动通信系统（如GSM）。

3. 波分复用（WDM, Wavelength Division Multiplexing）

（1）定义

波分复用是一种基于光波长的多路复用技术，主要用于光纤通信系统。它通过为每个信号分配不同的光波长来实现多路信号的同时传输。

（2）工作原理

  • 光纤中的可用频谱被划分为多个波长（颜色）。
  • 每个信号使用一个特定的波长进行传输。
  • 不同波长的光信号在同一根光纤中传输，彼此互不干扰。

（3）特点

  • 优点：
    • 频谱利用率极高，适合高速大容量通信。
    • 支持多路信号同时传输，干扰小。
    • 特别适合光纤通信的高带宽需求。
  • 缺点：
    • 设备成本较高，技术复杂。
    • 对光源和滤波器的精度要求高。

（4）应用场景

  • 光纤通信系统（如DWDM，密集波分复用）。
  • 数据中心互联和骨干网络。
  • 高速互联网服务提供商（ISP）。

4. 码分复用（CDM, Code Division Multiplexing）

（1）定义

码分复用是一种通过为每个信号分配唯一的编码序列来区分信号的技术。所有信号在同一时间、同一频段上传输，但通过编码实现信号的分离。

（2）工作原理

  • 所有信号在同一时间、同一频段上传输。
  • 每个信号通过独特的伪随机噪声（PN）码进行调制。
  • 接收端通过解码提取特定信号，其他信号被视为噪声。

（3）特点

  • 优点：
    • 频谱利用率高，支持更多信号同时传输。
    • 抗干扰能力强，信号隐蔽性好。
    • 适合动态变化的通信环境。
  • 缺点：
    • 实现复杂，需要高精度的同步和解码。
    • 多用户干扰（多址干扰）可能降低性能。

（4）应用场景

  • 移动通信系统（如CDMA2000、WCDMA）。
  • 军事通信系统（因其信号隐蔽性强）。
  • 卫星通信。

流量控制和拥塞控制
