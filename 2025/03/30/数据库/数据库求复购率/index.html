-- 2.统计用户三月份的回购率和复购率

^f104cf

统计复购率：

这段 SQL 查询的目的是统计在 order_info 表中，满足特定条件的订单数据，并进行分组和计数。以下是逐步分解和解释：

1. 子查询部分

  • 作用：
    • 从 order_info 表中筛选出满足以下条件的数据：
      • isPaid = "已支付"：订单状态为“已支付”。
      • month(paidTime) = 3：支付时间是 3 月份（假设 paidTime 是一个日期字段）。
    • 按照 userid 进行分组，并计算每个用户的订单数量（即 count(userid)），并将结果命名为 ct。
  • 输出结果：
    • 子查询会返回一个临时表 t，包含两列：
      • userid：用户 ID。
      • ct：该用户在 3 月份的已支付订单数量。

2. 主查询部分

  • 作用：
    • 主查询对子查询的结果进行进一步统计。
(1) count(ct)
  • 含义：
    • 统计子查询中所有用户的数量。
    • 因为 ct 是每个用户的订单数量，且子查询中每个用户都会有一个 ct 值，所以 count(ct) 实际上就是统计有多少个用户（即 userid 的总数）。
(2) count(if(ct > 1, 1, null))
  • 含义：
    • 使用 if 函数判断每个用户的订单数量是否大于 1：
      • 如果 ct > 1，则返回 1；
      • 否则返回 null。
    • count() 函数会忽略 null 值，因此这个表达式的作用是统计订单数量大于 1 的用户数量。

最终结果

主查询会返回两列： 1. count(ct)： - 总共有多少个用户在 3 月份有已支付的订单。 2. count(if(ct > 1, 1, null))： - 在这些用户中，有多少个用户的订单数量大于 1。

示例数据与结果

假设 order_info 表中有以下数据：

userid paidTime isPaid
1 2023-03-01 已支付
1 2023-03-05 已支付
2 2023-03-10 已支付
3 2023-03-15 已支付
3 2023-03-20 已支付
4 2023-04-01 已支付

执行上述查询后：

  1. 子查询结果：
userid ct
1 2
2 1
3 2
  1. 主查询结果：
count(ct) count(if(ct > 1, 1, null))
3 2
  • count(ct)：总共有 3 个用户（userid 为 1、2、3）。
  • count(if(ct > 1, 1, null))：其中 2 个用户（userid 为 1 和 3）的订单数量大于 1。
### 统计回购率
这段 SQL 查询的目的是分析用户的购买行为，特别是计算每个月的 购买人数 和 回购人数。以下是对这段代码的详细解析和优化建议。

查询的目标

  1. 购买人数：某个月有多少用户进行了支付。
  2. 回购人数：这些用户中，在前一个月也进行了支付的人数。

Step 1: 分析子查询和自连接

子查询部分

  • 作用：
    • 筛选出所有“已支付”的订单，并按 userid 和支付时间的月份（格式化为 YYYY-MM-01）分组。
    • date_format(paidTime, "%Y-%m-01") 将支付时间转换为每月的第一天，方便按月统计。
    • 结果是一个临时表，包含每个用户在每个月的支付记录。
    • group by可以起到去重的作用，因为本题只需要知道是否每月都有购买即可
  • 输出示例：

自连接部分

  • 作用：
    • 将同一个子查询结果分别命名为 t1 和 t2，并进行左连接。
    • 连接条件是：
      • 用户 ID 相同（t1.userid = t2.userid）。
      • t1 的月份是 t2 的月份的下一个月（t1.m = date_sub(t2.m, interval 1 month)）。
    • 这种连接方式可以匹配出用户在当前月和上个月的支付记录。

Step 2: 统计购买人数和回购人数

主查询部分

  • 作用：
    • 按照 t1.m（当前月）分组，统计每个月的购买人数和回购人数。
    • count(t1.m) 计算的是当前月的购买人数（即 t1 中的记录数）。
    • count(t2.m) 计算的是当前月的回购人数（即 t2 中有匹配记录的用户数）。
  • 输出示例：

关键点解析

  1. 为什么使用 LEFT JOIN？
    • 使用 LEFT JOIN 是为了确保即使某个用户在当前月没有前一个月的支付记录，也能保留当前月的数据。
    • 如果使用 INNER JOIN，则只会保留那些既有当前月又有前一个月支付记录的用户。
  2. 如何判断回购？
    • 回购的定义是：用户在当前月和前一个月都有支付记录。
    • 通过 t1.m = date_sub(t2.m, interval 1 month) 来实现月份的匹配。
  3. 性能优化建议：
    • 如果数据量较大，可以考虑对 order_info 表的 paidTime 和 ispaid 字段建立索引，以加速查询。
    • 避免重复计算子查询，可以通过创建临时表或视图来存储子查询结果。
