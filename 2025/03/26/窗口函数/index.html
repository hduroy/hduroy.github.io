窗口函数

发表于2025-03-26|更新于2025-03-26|专业课复习
|浏览量:

窗口函数（Window Function） 是 SQL 中一种强大的工具，用于在查询结果集中执行复杂的分析和计算操作。与传统的聚合函数（如 SUM()、AVG() 等）不同，窗口函数不会将结果集分组为单个输出行，而是保留每一行的详细信息，同时基于一组相关的行（称为“窗口”）进行计算。

1. 窗口函数的基本概念

1.1 什么是窗口？

  • 窗口 是指一个逻辑上的数据子集，通常通过某些条件（如分组或排序）定义。
  • 窗口函数在每个窗口内执行计算，但不会改变原始数据的结构。

1.2 窗口函数的特点

  1. 保留原始行：
    • 聚合函数（如 GROUP BY）会将多行数据压缩为一行。
    • 窗口函数则保留每一行的详细信息，同时提供额外的计算列。
  2. 灵活性高：
    • 可以在不改变表结构的情况下，执行复杂的排名、累计求和、移动平均等操作。
  3. 高效性：
    • 窗口函数避免了多次扫描表的需求，性能优于嵌套查询或自连接。

2.1 关键部分解释

  1. function_name：
    • 指定要使用的窗口函数，例如 ROW_NUMBER()、RANK()、SUM()、AVG() 等。
  2. OVER 子句：
    • 定义窗口的范围和行为。
    • 包括以下三个可选部分：
      • PARTITION BY：将数据分为多个窗口（类似于 GROUP BY 的作用）。
      • ORDER BY：指定窗口内的排序规则。
      • ROWS/RANGE：定义窗口的边界（如前 N 行、当前行等）。

4. 窗口函数的应用场景

4.1 排名

  • 使用 ROW_NUMBER()、RANK() 或 DENSE_RANK() 对数据进行排名。
  • 示例：按销售额对销售人员进行排名。

4.2 累计计算

  • 使用 SUM() 或 AVG() 计算累计值或移动平均值。
  • 示例：计算每日销售额的累计总和。

4.3 数据对比

  • 使用 LAG() 或 LEAD() 比较当前行与前后行的数据。
  • 示例：比较当前日与前一天的销售额差异。

4.4 数据分桶

  • 使用 NTILE() 将数据划分为若干组。
  • 示例：将学生成绩分为 4 个等级。

5. 窗口函数的优势

  1. 无需分组：
    • 窗口函数保留了每一行的详细信息，而不需要像 GROUP BY 那样压缩数据。
  2. 灵活的窗口定义：
    • 可以通过 PARTITION BY 和 ORDER BY 精确控制窗口的范围。
  3. 高效的复杂计算：
    • 窗口函数可以一次性完成排名、累计求和、移动平均等复杂操作，而不需要多次查询或临时表。

6. 窗口函数与聚合函数的区别

特性 窗口函数 聚合函数
是否分组 不分组，保留每一行 分组后返回单行
是否改变结果集结构 不改变，保留原始数据 改变，压缩为分组后的结果
应用场景 复杂分析（排名、累计、对比等） 总体统计（求和、平均值等）

7. 示例总结

假设有一个销售记录表 sales： | id | date | amount | |-----|------------|--------| | 1 | 2023-01-01 | 100 | | 2 | 2023-01-02 | 200 | | 3 | 2023-01-03 | 150 |

示例 1：累计销售额

结果： 窗口大小默认是从第一行到当前行，所以可以进行一个累加运算

id date amount cumulative_amount
1 2023-01-01 100 100
2 2023-01-02 200 300
3 2023-01-03 150 450

示例 2：排名

结果： 排名不需要窗口大小

id date amount rank
2 2023-01-02 200 1
3 2023-01-03 150 2
1 2023-01-01 100 3

  • RANK() 不需要窗口范围 ：

    • 它只依赖于 ORDER BY 子句定义的排序规则，而不是窗口内的具体行。
    • 即使你显式指定窗口范围（如 ROWS BETWEEN ... AND ...），也不会影响 RANK() 的结果。

  • 窗口范围适用于聚合函数 ：

    • 对于 SUM()、AVG() 等聚合函数，窗口范围非常重要，因为它决定了计算所涉及的行
