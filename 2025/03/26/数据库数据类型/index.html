数据库数据类型

发表于2025-03-26|更新于2025-03-26|专业课复习
|浏览量:

在数据库中，数据类型用于定义列（字段）可以存储的数据种类。选择正确的数据类型不仅可以提高存储效率，还能优化查询性能。以下是常见数据库系统（如 MySQL、PostgreSQL、SQL Server 等）中常用的数据类型分类及其说明：

1. 数值类型

数值类型用于存储整数或浮点数。

1.1 整数类型

  • TINYINT
    • 存储范围：-128 到 127（有符号），0 到 255（无符号）。
    • 用途：用于存储小范围的整数，如布尔值（0 或 1）。
  • SMALLINT
    • 存储范围：-32,768 到 32,767（有符号），0 到 65,535（无符号）。
    • 用途：用于存储较小的整数。
  • MEDIUMINT
    • 存储范围：-8,388,608 到 8,388,607（有符号），0 到 16,777,215（无符号）。
    • 用途：MySQL 特有的中间范围整数类型。
  • INT / INTEGER
    • 存储范围：-2,147,483,648 到 2,147,483,647（有符号），0 到 4,294,967,295（无符号）。
    • 用途：最常用的整数类型。
  • BIGINT
    • 存储范围：-9,223,372,036,854,775,808 到 9,223,372,036,854,775,807（有符号）。
    • 用途：用于存储非常大的整数。

1.2 浮点数和定点数类型

  • FLOAT
    • 单精度浮点数，占用 4 字节。
    • 用途：用于存储近似的小数，适合对精度要求不高的场景。
  • DOUBLE / REAL
    • 双精度浮点数，占用 8 字节。
    • 用途：用于存储更高精度的小数。
  • DECIMAL / NUMERIC
    • 定点数类型，精确存储小数。
    • 格式：DECIMAL(M, D)，其中 M 是总位数，D 是小数位数。
    • 用途：用于存储货币金额等需要高精度的数值。

2. 字符串类型

字符串类型用于存储文本数据。

2.1 定长字符串

  • CHAR(n)
    • 固定长度的字符串，最大长度为 n（通常不超过 255）。
    • 如果实际内容长度小于 n，会用空格填充。
    • 用途：存储固定长度的字段（如邮政编码、身份证号）。

2.2 可变长字符串

  • VARCHAR(n)
    • 可变长度的字符串，最大长度为 n（通常不超过 65,535）。
    • 不会用空格填充，节省存储空间。
    • 用途：存储长度不固定的文本（如姓名、地址）。

2.3 大文本类型

  • TEXT
    • 存储大段文本，最大长度为 65,535 字节。
    • 用途：存储文章、描述等较长的文本。
  • MEDIUMTEXT
    • 最大长度为 16,777,215 字节。
    • 用途：存储更大的文本数据。
  • LONGTEXT
    • 最大长度为 4GB。
    • 用途：存储超大文本数据。

3. 日期和时间类型

日期和时间类型用于存储日期或时间相关的数据。

  • DATE
    • 格式：YYYY-MM-DD。
    • 范围：1000-01-01 到 9999-12-31。
    • 用途：存储日期。
  • TIME
    • 格式：HH:MM:SS。
    • 范围：-838:59:59 到 838:59:59。
    • 用途：存储时间。
  • DATETIME
    • 格式：YYYY-MM-DD HH:MM:SS。
    • 范围：1000-01-01 00:00:00 到 9999-12-31 23:59:59。
    • 用途：存储日期和时间。
  • TIMESTAMP
    • 格式：YYYY-MM-DD HH:MM:SS。
    • 范围：1970-01-01 00:00:01 UTC 到 2038-01-19 03:14:07 UTC。
    • 用途：存储时间戳，常用于记录事件发生的时间。
  • YEAR
    • 格式：YYYY 或 YY。
    • 范围：1901 到 2155。
    • 用途：存储年份。

4. 二进制类型

二进制类型用于存储非文本数据（如图片、文件等）。

  • BINARY(n)
    • 固定长度的二进制数据，最大长度为 n。
    • 用途：存储固定大小的二进制数据。
  • VARBINARY(n)
    • 可变长度的二进制数据，最大长度为 n。
    • 用途：存储可变大小的二进制数据。
  • BLOB
    • 存储大块二进制数据，最大长度为 65,535 字节。
    • 用途：存储图片、音频、视频等。
  • MEDIUMBLOB
    • 最大长度为 16,777,215 字节。
    • 用途：存储更大的二进制数据。
  • LONGBLOB
    • 最大长度为 4GB。
    • 用途：存储超大的二进制数据。

5. 枚举和集合类型

5.1 ENUM

  • 定义一组固定的值，列的值只能是其中一个枚举值。
  • 示例：ENUM('Male', 'Female')。
  • 用途：用于存储有限选项的字段（如性别、状态）。

5.2 SET

  • 允许从一组固定的值中选择多个值，值之间用逗号分隔。
  • 示例：SET('A', 'B', 'C')。
  • 用途：用于存储多选字段（如兴趣爱好）。

6. JSON 类型

  • JSON
    • 用于存储 JSON 格式的数据。
    • 支持嵌套结构，可以高效地存储和查询复杂数据。
    • 用途：存储动态结构化数据（如配置信息、用户属性）。

7. 布尔类型

  • BOOLEAN / BOOL
    • 实际上是 TINYINT(1) 的别名，0 表示 FALSE，1 表示 TRUE。
    • 用途：存储布尔值（如是否启用、是否完成）。

8. 其他特殊类型

  • UUID
    • 用于存储通用唯一标识符（Universally Unique Identifier）。
    • 用途：生成唯一 ID，避免主键冲突。
  • ARRAY（部分数据库支持，如 PostgreSQL）
    • 存储数组类型的数据。
    • 用途：存储列表形式的数据。

总结

选择合适的数据类型非常重要，以下是一些建议： 1. 优先考虑存储需求：根据数据的实际范围选择最小的数据类型。 2. 关注性能：定长类型（如 CHAR）比可变长类型（如 VARCHAR）更快，但可能浪费存储空间。 3. 避免过度设计：不要为未来可能的需求预留过多的存储空间。 4. 使用现代特性：如 JSON 类型或 UUID，以简化复杂数据的存储和管理。

如果你有具体的场景或问题，欢迎进一步提问！
