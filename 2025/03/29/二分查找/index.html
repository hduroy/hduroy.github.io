二分查找

发表于2025-03-29|更新于2025-03-29|数据结构
|浏览量:

二分查找是一种高效的查找算法，时间复杂度为 O(log n)。然而，二分查找的实现中，边界条件往往是容易出错的地方。理解二分查找的边界条件对于正确实现算法至关重要。

以下是对二分查找边界条件的详细分析，包括常见问题和解决方法。

1. 基本思想

二分查找的核心思想是： - 在一个有序数组中，通过比较目标值与中间值，逐步缩小搜索范围。 - 如果目标值小于中间值，则在左半部分继续查找；如果大于中间值，则在右半部分继续查找。

2. 常见实现方式

（1）左闭右闭区间 [left, right]

这是最常用的实现方式，搜索范围包括两端点 left 和 right。

边界条件：

  • 初始范围：[0, nums.size() - 1]。
  • 循环条件：left <= right，即当 left > right 时结束循环。
  • 更新规则：
    • left = mid + 1：排除中间值及其左侧。
    • right = mid - 1：排除中间值及其右侧。

（2）左闭右开区间 [left, right)

在这种实现中，搜索范围不包括右端点 right。

边界条件：

  • 初始范围：[0, nums.size())。
  • 循环条件：left < right，即当 left == right 时结束循环。
  • 更新规则：
    • left = mid + 1：排除中间值及其左侧。
    • right = mid：排除中间值及其右侧。

3. 边界条件的理解

（1）为什么使用 left <= right 或 left < right？

  • left <= right：适用于左闭右闭区间 [left, right]，表示搜索范围包括两端点。
  • left < right：适用于左闭右开区间 [left, right)，表示搜索范围不包括右端点。

选择哪种方式取决于你的实现习惯和具体需求。两种方式都能正确实现二分查找，但需要确保更新规则与循环条件一致。

（2）为什么更新 left 和 right 时要加减 1？

  • left = mid + 1：因为 mid 已经被检查过，且当前分支已经确定目标值不可能等于 mid。
  • right = mid - 1：同理，mid 已经被检查过，目标值不可能等于 mid。

如果不加减 1，可能会导致死循环。例如，当 left == right 且未找到目标值时，搜索范围无法缩小。

（3）为什么计算 mid 时使用 (left + (right - left) / 2)？

  • 防止溢出：(left + right) / 2 可能导致整数溢出（特别是在 left 和 right 很大的情况下）。
  • 等价性：(left + (right - left) / 2) 等价于 (left + right) / 2，但更安全。

4. 特殊情况处理

（1）空数组

如果输入数组为空，直接返回 -1 或其他表示未找到的结果。

（2）目标值不存在

如果目标值不在数组中，循环结束时返回 -1。

5. 变体：查找第一个或最后一个目标值的位置

6. 总结

  • 核心思想：通过比较中间值与目标值，逐步缩小搜索范围。
  • 边界条件：
    • 左闭右闭区间 [left, right] 使用 left <= right。
    • 左闭右开区间 [left, right) 使用 left < right。
  • 更新规则：
    • left = mid + 1 排除左侧。
    • right = mid - 1 排除右侧。
  • 特殊情况：
    • 空数组、目标值不存在、多个相同目标值的情况。
